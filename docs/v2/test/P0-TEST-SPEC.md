# P0 测试规格说明

基于源码分析的准确测试规格。

## T01: 应用启动

**目标**: 验证应用能正常启动并显示基本 UI

**前置条件**: 无

**测试步骤**:
1. 访问 `http://localhost:5173`
2. 等待页面加载完成
3. 处理可能出现的更新日志弹窗（按 ESC 关闭）

**验证点**:
- [ ] 页面标题为 "onmyoji-flow"
- [ ] 工具栏可见（包含"文件"、"设置"、"帮助"按钮）
- [ ] 组件面板可见（显示"组件库"标题）
- [ ] 画布区域可见
- [ ] 属性面板可见（右侧）
- [ ] 无 JavaScript 错误

**选择器**:
- 工具栏: `.toolbar`
- 文件按钮: `button:has-text("文件")`
- 组件面板: 包含 `heading:has-text("组件库")`
- 更新日志弹窗: `.el-dialog` 包含 `heading:has-text("更新日志")`

---

## T02: 节点创建（拖拽）

**目标**: 验证从组件面板拖拽节点到画布的基本功能

**前置条件**: 应用已启动

**测试步骤**:
1. 打开应用
2. 关闭更新日志弹窗（如果出现）
3. 在组件面板找到"长方形"组件
4. 拖拽"长方形"到画布中心
5. 释放鼠标

**验证点**:
- [ ] 画布中出现一个长方形节点
- [ ] 节点可见且有图形渲染
- [ ] 节点数量 > 0

**选择器**:
- 长方形组件: `text=长方形` (在组件面板中)
- 画布: `.lf-canvas-overlay` 或 `[class*="canvas"]`
- 节点: `.lf-node` 或 `[class*="node"]`

---

## T03: 多 Tab 管理

**目标**: 验证多 Tab 功能，确保每个 Tab 的数据独立

**前置条件**: 应用已启动

**测试步骤**:
1. 打开应用（默认 Tab: "File 1"）
2. 在 Tab A 创建一个长方形节点
3. 记录 Tab A 的节点数量
4. 查找并点击"新建 Tab"按钮（可能在 Tab 栏右侧）
5. 验证现在有 2 个 Tab
6. 在 Tab B 创建一个圆形节点
7. 记录 Tab B 的节点数量
8. 切换回 Tab A

**验证点**:
- [ ] Tab A 有长方形节点
- [ ] Tab B 有圆形节点
- [ ] 两个 Tab 的数据互不干扰
- [ ] Tab 切换正常工作

**选择器**:
- Tab 项: `.el-tabs__item` 或 `[role="tab"]`
- 新建 Tab 按钮: `.el-tabs__new-tab` (Element Plus editable 属性自动生成)
- 圆形组件: `text=圆形`

**注意**: 
- Element Plus 的 `editable` 属性会在 tabs 右侧自动生成加号按钮
- Tab 切换后需要等待 1-1.5 秒让画布完全重新渲染

## T04: 数据持久化（localStorage）

**目标**: 验证画布数据能保存到 localStorage 并在刷新后恢复

**前置条件**: 应用已启动

**测试步骤**:
1. 打开应用
2. 创建一个长方形节点
3. 记录节点数量（应该 > 0）
4. 刷新页面（`page.reload()`）
5. 等待页面加载完成
6. 关闭更新日志弹窗
7. 检查画布中的节点数量

**验证点**:
- [ ] 刷新前节点数量 > 0
- [ ] 刷新后节点数量与刷新前相同
- [ ] 节点位置和类型保持一致

**注意**: 
- 需要等待足够时间让数据保存到 localStorage
- 刷新后需要重新处理更新日志弹窗

---

## T05: 导入/导出

**目标**: 验证数据导出和导入功能

**前置条件**: 应用已启动

**测试步骤**:
1. 打开应用
2. 点击"文件"菜单
3. 点击"加载示例"（加载预设数据）
4. 等待数据加载完成
5. 记录节点数量（应该 > 0）
6. 点击"文件"菜单 → "导出"
7. 下载 JSON 文件
8. 点击"清空画布"按钮
9. 验证画布为空（节点数量 = 0）
10. 点击"文件"菜单 → "导入"
11. 选择刚才导出的 JSON 文件
12. 等待导入完成
13. 检查节点数量

**验证点**:
- [ ] 加载示例后节点数量 > 0
- [ ] 导出成功（文件下载）
- [ ] 清空画布后节点数量 = 0
- [ ] 导入后节点数量恢复到导出前的值

**选择器**:
- 文件菜单: `button:has-text("文件")`
- 加载示例: `.el-dropdown-menu__item` 过滤包含"加载示例"文本
- 导出: `.el-dropdown-menu__item` 过滤包含"导出"文本
- 清空画布: `button:has-text("清空画布")`
- 导入: `.el-dropdown-menu__item` 过滤包含"导入"文本

**注意**: 
- 所有菜单项都在文件菜单的下拉列表中，需要先点击"文件"按钮
- 每次操作菜单项前都需要重新打开文件菜单
- 需要设置下载监听器
- 导入需要触发文件选择对话框

## 通用注意事项

### 更新日志弹窗处理
- **触发条件**: `localStorage.getItem("appVersion") !== currentAppVersion`
- **关闭方式**: 按 ESC 键（最可靠）
- **选择器**: `.el-dialog` 包含 `heading:has-text("更新日志")`
- **等待时间**: 弹窗出现需要约 1 秒

### 等待策略
- 页面加载: `waitForLoadState('domcontentloaded')`
- 弹窗出现: `waitForTimeout(1000)` 后检查
- 节点创建: 拖拽后 `waitForTimeout(500)`
- 数据加载: 操作后 `waitForTimeout(1000-1500)`

### 选择器优先级
1. 优先使用文本选择器: `text=长方形`
2. 其次使用类名: `.lf-canvas-overlay`
3. 最后使用属性: `[role="tab"]`

### 测试隔离
- 每个测试独立运行
- 测试前可选择清空 localStorage（模拟新用户）
- 使用 `--workers=1` 顺序执行避免冲突
